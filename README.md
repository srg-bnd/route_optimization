# Оптимизация маршрутов

Логистическая компания столкнулась с проблемой планирования маршрутов для своего автопарка. Маршрут представляет собой путь между городами, который должен удовлетворить нескольким критериям одновременно:

- быть коротким по протяжённости,
- быстрым по времени (например, из-за ограничений в скорости),
- дешёвым по стоимости (например, из-за платных дорог).

Вам необходимо разработать систему, которая по заданной карте дорог и требованиям найдёт не один, а набор маршрутов между двумя городами, каждый из которых является оптимальным по одному из критериев. На финальном этапе система должна выбрать единственный компромиссный маршрут, используя заданные приоритеты.

## Входные данные

Программа получает входные данные из файла “input.txt”.
Данные разделены на несколько секций.

- Секция [CITIES] содержит список всех городов, по одному на строку.
Каждая строка имеет формат: ID: Название_города
➔ ID — натуральное число;
➔ Название_города — строка, которая может содержать пробелы.
- Секция [ROADS] содержит описание дорог между городами.
Каждая строка имеет формат: ID1 -ID2: длина(км), время(мин), Стоимость(руб).
Все три параметра — целые положительные числа. Дороги двусторонние.
- Секция [REQUESTS] содержит запросы на построение маршрута.
Каждая строка имеет формат: Город_отправления -> Город_назначения | Приоритеты (Д,В,С).
Город_отправления и Город_назначения — это строковые значения названий городов (не ID).
Приоритеты — это строка из трех символов Д (длина), В (время), С (стоимость), разделенных запятыми и заключенных в общие скобки. Порядок символов определяет их важность. Например, (Д,С,В) означает, что длина — самый важный критерий, затем стоимость, и время — наименее важный.

### Пример входных данных

- [CITIES]
- 1: Москва
- 2: Санкт-Петербург
- 3: Нижний Новгород
- 4: Казань
- [ROADS]
- 1 - 2: 700, 480, 800
- 1 - 3: 400, 250, 300
- 2 - 3: 1100, 700, 1200
- 3 - 4: 350, 300, 500
- 1 - 4: 800, 600, 1000
- [REQUESTS]
- Москва -> Санкт-Петербург | (Д,В,С)
- Нижний Новгород -> Казань | (С,В,Д)

## Выходные данные

Для каждого запроса из секции [REQUESTS] программа должна вывести в файл “output.txt” следующие данные:

- Три оптимальных маршрута по отдельности по каждому критерию.
Формат вывода для одного маршрута: КРИТЕРИЙ: Город1 -> Город2 -> … -> ГородN | Параметр1=Значение1, Параметр2=Значение2, Параметр3=Значение3.
КРИТЕРИЙ — это ДЛИНА, ВРЕМЯ или СТОИМОСТЬ.
Значения параметров выводятся для всех трех критериев.
- Итоговый компромиссный маршрут, выбранный на основе приоритетов.
Формат вывода: КОМПРОМИСС: Город1 -> Город2 -> … -> ГородN | Д=…, В=…, С=…

### Пример выходных данных для первого запроса

- ДЛИНА: Москва -> Санкт-Петербург | Д=700, В=480, С=800
- ВРЕМЯ: Москва -> Санкт-Петербург | Д=700, В=480, С=800
- СТОИМОСТЬ: Москва -> Санкт-Петербург | Д=700, В=480, С=800
- КОМПРОМИСС: Москва -> Санкт-Петербург | Д=700, В=480, С=800

(В данном примере все маршруты совпали, так как есть прямая дорога, оптимальная по всем параметрам)
